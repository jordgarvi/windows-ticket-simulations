# User Offboarding & Access Revocation

## Purpose

Provide a structured first-line offboarding workflow for securely disabling user accounts and revoking organisational access following employee departure.

---

## Scope

This procedure applies to:
- Employee departures  
- Role changes requiring access removal  
- Temporary account deactivation  
- Security-related access revocation requests  

---

## Offboarding Objectives

The purpose of offboarding is to:
- Prevent continued access following departure  
- Revoke unnecessary permissions  
- Protect organisational resources  
- Maintain accurate access records  
- Support secure user lifecycle management  

---

## Pre-Offboarding Verification

Before any access changes are performed:

- Verify the offboarding request originated from HR or authorised management  
- Confirm manager approval has been received  
- Confirm the employee's final working date  
- Review timing requirements for account disablement  

Access revocation timing is operationally important.

In standard departures, access is typically revoked at the end of the employee's final working day.

In involuntary departure scenarios, immediate access revocation may be required for security reasons.

---

## Access Review Procedure

Before revoking access, review:
- User account status  
- Shared folder permissions  
- Departmental resource access  
- Mapped drive access  
- Assigned workstation or company equipment  

This helps ensure all assigned access is identified before removal activities begin.

---

## Account Disablement Procedure

User accounts should normally be:
✅ disabled
NOT:
❌ immediately deleted

Disabling accounts:
- Preserves audit history  
- Supports data retention review  
- Allows recovery if required  
- Reduces risk of accidental permanent deletion  

---

## Access Revocation Procedure

Remove:
- Shared folder permissions  
- Departmental resource access  
- Mapped drive access  
- Authentication access where applicable  

Access removal should follow least privilege principles to ensure no unnecessary access remains active.

---

## Data Handling Considerations

Operational offboarding activities may also require review of:
- Mailbox retention or forwarding requirements  
- Shared departmental files owned by the user  
- OneDrive or personal work data retention  
- Ongoing business access requirements for retained data  

These activities should follow organisational data retention procedures.

---

## Equipment & Asset Review

As part of offboarding activities:
- Review company equipment return status  
- Confirm return of laptops, access cards, or assigned devices  
- Update asset records where appropriate  

Offboarding includes both digital and physical access management.

---

## Verification Procedure

Following access revocation:
- Confirm user account disabled successfully  
- Verify authentication no longer possible  
- Confirm departmental permissions removed  
- Test previously assigned shared resource access  
- Review for residual access  

Attempting to access previously assigned resources after revocation can help verify offboarding activities completed successfully.

---

## Security Considerations

Key offboarding security considerations include:
- Verifying authorisation before disabling accounts  
- Revoking access promptly following departure  
- Removing shared resource permissions  
- Reviewing residual access during verification  
- Documenting completed offboarding actions appropriately  

These activities help reduce the risk of unauthorised access following employee departure.

---

## Escalation Guidance

Escalate the offboarding request if:
- Conflicting instructions are received from HR or management  
- Unusual account activity is identified during review  
- Shared business-critical data ownership requires clarification  
- Security concerns are identified during offboarding activities  

Further escalation may require management review or security team involvement.
